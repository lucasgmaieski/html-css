Container Cascata

Efeito cascata

  1. Primeiro ele vai ler o link para o css lá em cima no head(estilo externo)
  2. Dentro do css ele vai seguir lendo e se tiver dois comandos iguais vai ser usado o ultimo
  3. Depois ele volta e lê o style dentro do head e aplica(estilo interno)
  4. E por fim vai aplicar o style que tiver em alguma tag(estilo inline)

Container Especificidade

Quanto mais específico for o seletor maior a príoridade. Não importa a ordem que esteja

  • O id é o seletor com maior prioridade só perdendo para o um atributo style inline
  • !important pode ser usado para dar prioridade máxima tbm mas deve ser evitado por que dificulta depois na hora de fazer a manutanção no codigo

Fontes

  • Serif = mais usada em jornais e revistas impressos, teas perninhas
  • Sanserif = muito usada na internet pois facilita a leitura
  • monospace = mesma largura das palavras independente de maiuscula ou minuscula
  • Nomes de fontes compostos devem estar entre aspas simples ex: 'Courier New'
  • font-weight de 100 até 400= normal e regular
    do 500 ao 700= bold
    do 700 ao 900= extra bold
  • Na hora de baixar um fonte atente-se para os glifos se tem tudo que você quer

Display

  • Display inline não da pra mudar altura nem largura
  • Display inline-block permite colocar algura e largura
  • Display none esconde os elementos, não some do html. Usado pra esconder elementos muito grandes em dispositivos mobile

Position

  • Static é o padrão que não se mexe
  • Relative deixa o elemento pra ser posisionado em relação ao lugar onde foi criado
  • Absolute cria uma segunda camada(dimensão) para o elemento
    Para manter esse elemento onde ele foi criado e facilitar a movimentação podemos colocar ele dentro de outro elemento pai com posicionamento relativo. O elemento absolut sempre vai se movimentar em relação ao seu pai relative
  • Fixed também cria uma nova camada por cima mas fica fixo naquele lugar da tela mesmo quando vai pra cima ou pra baixo com scroll
